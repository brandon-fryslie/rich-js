# Doc Spec: Markdown

The markdown doc explains the Markdown renderable for rendering Markdown-formatted text in the terminal.

## Sections

### Basic usage

Show constructing a Markdown object with a markdown string and printing it. Use a short example that includes a heading, emphasized text, a list, and a code block — these are the elements most likely to appear in real use.

### Code blocks

Note that code blocks inside markdown are rendered with syntax highlighting.

### Practical pattern

Show reading a Markdown file from disk and rendering it. This is the most common real-world use.

## Constraints

- Keep this doc short — Markdown is a simple renderable
- The example must show enough markdown elements that the reader can see what the terminal rendering looks like (headings, lists, code block)
- Do not document every Markdown element — the reader already knows Markdown; show what the terminal rendering looks like
